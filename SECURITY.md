# 安全政策

## 支持版本

目前仅维护最新主版本和最近一个稳定版本。请优先升级到最新版本后再反馈问题。

| 版本 | 支持 |
| --- | --- |
| 2.2.4 | ✅ |
| 2.1.10 | ✅ 仅安全修复 |
| 2.0.x | ❌ |
| < 2.0 | ❌ |

## 报告漏洞

如果你发现安全漏洞，请不要公开提交 Issue，也不要在群聊或公开页面贴出漏洞细节。

请通过以下方式私下联系维护者：

- GitHub：通过仓库所有者主页联系
- 或提交一个不包含漏洞细节的 Issue，标题写：`安全漏洞报告请求联系`

报告时请尽量包含：

- 受影响版本
- 漏洞类型
- 复现步骤
- 影响范围
- 是否已被公开或利用

## 处理流程

收到报告后，我会尽快确认问题。如果漏洞成立，会优先发布安全更新。

一般处理流程：

1. 确认漏洞是否存在
2. 评估影响范围
3. 修复并打包新版本
4. 发布安全更新
5. 在不泄露利用细节的前提下说明修复内容

## 注意事项

请不要提交或公开以下内容：

- 管理员密码
- 更新密码
- JWT 密钥
- SSH 密钥
- 服务器 IP、端口和后台地址
- 真实库存数据
- 用户数据
- 可直接利用的攻击脚本
